# 套装系统地基：装备累积 + 套装计数（②b-1）设计

> Story: set-bonus-foundation（装备 epic ②b 套装效果，第一期 ②b-1）
> 日期：2026-06-26
> 引擎：Godot 4.6.3 / GDScript / GdUnit4
> 前置：②a 装备地基（9 槽 + 5 稀有度 + 31 件，HEAD=6e36549，已 push origin/main）

> **②b 三期拆分（本稿仅 ②b-1）**：
> - **②b-1 累积地基（本稿）**：8 套数据 + 累积获取系统 + 套装计数器 + 获取 UI（人形纸娃娃）+ 阶段/存档。
>   本期套装能集齐、能看件数与「已激活 3/6/9」标记，但**不产生任何战斗加成**。
> - **②b-2 效果引擎 + 低风险 6 套**：套装效果应用框架 + 铁壁/狂战/医者/航海/嗜血/荆棘/处决 的 3/6/9 效果 + 激活预览。
> - **②b-3 高风险机制**：寒霜（新 debuff status）等需新增战斗机制的阶梯。

---

## 1. 概述（Overview）

把 ②a 的「一人最多 2 件、招募时一次性锁定」升级为**可随 run 推进逐步累积、能凑出套装的装备成长系统**。船员最多持 9 件（每槽 1 件）；起始/招募各直发 3 件（80% 同套），每场战斗后出战存活者各从 8 件候选（80% 偏向其主套）中选至多 2 件。新增**套装计数器**（每套件数 / 主套 / 已激活档位 3/6/9）供偏向逻辑、纸娃娃与未来效果引擎共用。本期**只铺数据 + 累积 + 计数 + UI + 存档**，套装效果留 ②b-2/3。

## 2. 玩家体验（Player Fantasy）

身为船长，麾下每名船员都在一整条航程里慢慢武装成型——起步就有一身初具雏形的套装，每打赢一场就能从缴获里挑两件，越凑越齐。摊开船员的人形装备面板，一眼看清主武器到项链九个部位穿了什么、什么品阶（颜色）、属于哪套，以及「铁壁 6/9」这样的集齐进度。集齐套装的成就感在本期先以「看得见的进度」兑现，套装真正发威是下一步的事。

## 3. 详细规则（Detailed Rules）

### 3.1 八大套装与装备数据
- 新增 **8 个套装**，每个 `set_id` 唯一：
  | set_id | 中文 | 主题（②b-2/3 效果方向） |
  |---|---|---|
  | `set_ironwall` | 铁壁 | 防御/减伤 |
  | `set_berserker` | 狂战 | 输出/附加伤害 |
  | `set_healer` | 医者 | 续航/自愈 |
  | `set_navigator` | 航海 | 战术/团队加成 |
  | `set_bloodthirst` | 嗜血 | 吸血 |
  | `set_thorns` | 荆棘 | 反伤 |
  | `set_executioner` | 处决 | 斩杀 |
  | `set_frost` | 寒霜 | 控制（新机制，②b-3） |
- **每套 9 件，每槽各 1 件**（主武器/副武器/头/护甲/手/腿/靴/戒指/项链）→ 共 **72 件**。本期装备池**全部属于某套**（取消无套装散件，简化 80% 偏向）。现有 31 件**重新归入八套**（填 `set_id`、按需调 `slot`），不足部分补足到 72。
- 每件保留独立 `rarity`（决定颜色）。套内 9 件按槽呈稀有度梯度（保证颜色多样、整套强度均衡）；纯数值增量（血/攻/射程/移动）沿用 ②a 既有量级。**精确数值表见实现期生成器输入表（plan 评审）**。
- 装备仍为只读静态模板，由 `EquipmentDataManager` 扫描缓存；运行时不写回。

### 3.2 容量
- 每船员最多 **9 件、每槽 1 件**。运行时存储 `RunManager._roster_equipment[crew_id] = {slot:int → eid}`（②a 已是此结构，天然支持 9 槽）。
- 敌方/无装备单位为空字典（不变）。

### 3.3 起始 & 招募：直发 3 件（80% 同套）
- `start_run` 给每名**起始船员**、`confirm_recruit` 给**新兵**：滚 **3 件并直接装上**，无交互。
- **80% 同套规则**：掷一次 d100（走 `_rng`）：
  - `< 80`：选一个锚定 `set_id`，从该套取 **3 件不同槽**。
  - 否则：取 **3 件随机套装的不同槽**装备（混搭）。
- 取消 ②a 的「招募滚 8 选 2 / 恰好选 2」规则（被直发 3 件取代）。`_pending_recruit_equip` 及相关 UI 移除。

### 3.4 战后：滚 8 选 2（仅出战存活者，80% 偏向主套）
- 每场**非末岛**战斗胜利后，对**本场出战且存活**的船员各滚 **8 件候选**，玩家每人**选至多 2 件**（可少选/跳过）。
- **80% 偏向规则**：8 件**逐件独立**掷 d100：
  - `< 80`：取该船员**主套**装备——优先其**尚未拥有的空槽**件；若主套对应空槽已无可补，则取主套任意件。
  - 否则：取随机套装/随机槽装备。
  - 无装备（无主套）者：8 件全随机。
- **主套判定**：`get_dominant_set` = 持有件数最多的 `set_id`；**并列取字典序最小**（确定性）；无装备返回 `""`。
- **选取与槽冲突**：
  - 选中件的槽为**空** → 直接装上。
  - 选中件的槽**已占** → 弹替换确认；确认则**丢弃旧件**装新件，取消则不选。
  - 同一次最多选 2 件；两件同槽时第二件按替换处理（针对同一槽）。
- 末岛胜利不触发补装（直接 RUN_END，沿用现有）。

### 3.5 套装计数器（纯函数，本期只读用途）
挂在 RunManager，读 `_roster_equipment` + 各 eid 的 `set_id`：
- `get_set_counts(crew_id) -> Dictionary`：`{set_id → count}`（仅含 count≥1 的套）。
- `get_dominant_set(crew_id) -> String`：件数最多的 set_id，并列取字典序最小，无装备返回 `""`。
- `get_active_set_tier(crew_id, set_id) -> int`：返回 ≤ 该套件数的最大阈值 ∈ {0,3,6,9}。
- 本期仅供：3.4 的 80% 偏向、纸娃娃显示「铁壁 6/9」。②b-2 效果引擎直接复用，**本期不接战斗逻辑**。

### 3.6 阶段接缝
- 新增阶段 `RunPhase.RUN_EQUIPPING`（字符串 `"EQUIPPING"`）。
- `_on_battle_won`（非末岛分支）：先为全体出战存活者滚好 8 件候选写入 `_pending_battle_equip = {crew_id → [8 eids]}`（走 `_rng`），再 `_set_run_phase(RUN_EQUIPPING)` 并 `_goto_route.call()`。
- 玩家逐人选取：每选定 1 件即调 `equip_piece` 写入 `_roster_equipment`；该船员选满 2 件或主动结束 → 从 `_pending_battle_equip` 移除该 crew_id。
- `_pending_battle_equip` 清空 → `_set_run_phase(RUN_RECRUITING)`（接回现有链：RECRUITING→CHARTING→…）。
- **预滚而非临屏滚**：补装为交互步骤，进入阶段时一次性滚定并存盘，保证存档中途退出后重进复现同样候选（仿 ②a `_pending_recruit_equip`）。

### 3.7 获取 UI（白盒 + 人形纸娃娃）
- **稀有度配色常量**（集中一处共享，供纸娃娃/卡面复用）：普通=灰白 / 稀有=蓝 / 史诗=紫 / 稀世=橙 / 传奇=红。
- **招募**：仍 3 候选选 1（现有）；选定后弹「获得 3 件装备」通知卡 + 纸娃娃预览。
- **战后补装屏**（RouteScene `EQUIPPING` 分支）：对每名出战存活船员依次：
  - 候选区：8 件按钮，文案「名字〔套装·稀有度〕+增量摘要」，按钮文字以稀有度色标注。
  - **人形纸娃娃（右）**：9 槽逐格显示当前装备（名字 + 稀有度色 + 套装标签），空槽显「空」；顶部列出已激活套装档位（如「铁壁 6/9」）。
  - 玩家选至多 2 件；选中已占槽 → 替换确认。
  - 全程白盒（程序化 Label/Button/Panel，无美术资源）。

## 4. 公式（Formulas）

- **同套滚（3 件）**：`r = _rng.randi() % 100`；`if r < SAME_SET_CHANCE(=80)` → anchor set 取 3 不同槽，`else` 混搭 3 不同槽。
- **偏向滚（每件）**：`r = _rng.randi() % 100`；`if r < BIAS_CHANCE(=80) and dominant != ""` → 主套（优先空槽件），`else` 随机。
- **主套**：`argmax_{set_id} count`，并列取 `min(set_id)`（字典序）。
- **激活档位**：`tier = 0; for t in [9,6,3]: if count >= t: tier = t; break`。
- 数值增量仍按 ②a：`有效值 = 基值 + Σ(已装槽增量)`，下限钳 0（`UnitInstance.get_*` 不变）。

## 5. 边界情况（Edge Cases）

- **装备池某槽/某套抽空**：滚装目标子池为空时优雅降级——同套滚降级为混搭；偏向滚降级为随机；仍空则该候选取任意可用件（确保 8/3 件数量满足，除非全池更少，理论不会）。
- **船员已满 9 槽**：战后仍可滚 8 选 2，但只能走「替换」；玩家可全跳过。
- **并列主套**：字典序最小，保证确定性与可测。
- **存档中途退出**：`_pending_battle_equip` + 当前 `RUN_EQUIPPING` 阶段持久化；重进复现同候选、续选未完成船员。
- **旧档兼容**：旧存档无 `_pending_battle_equip` 键 / 无 `EQUIPPING` 阶段 → 默认空、阶段降级（不崩）。`roster_equipment` 嵌套结构与 ②a 一致。
- **permadeath**：阵亡船员移出 roster 时其装备账本一并擦除（沿用 ②a），不进入战后补装。
- **末岛胜**：不进补装，直接 RUN_END。

## 6. 依赖（Dependencies）

- `EquipmentDefinition`（②a：含 `set_id` 字段，本期填值）/ `EquipmentDataManager`（②a：扫描缓存 + 按套/稀有度查询；本期 72 件数据）。
- `RunManager`（招募/战后/存档/阶段链）/ `UnitInstance`（有效值访问器，不改）。
- `RouteScene`（白盒中枢，加 EQUIPPING 分支）/ `SceneManager` / `EventBus`（battle_won / run_phase_changed）。
- `MapDataManager`/`TurnManager`/`BattleResolution` **不涉及**（本期无战斗效果）。

## 7. 可调旋钮（Tuning Knobs）

- `SAME_SET_CHANCE = 80`（起始/招募 3 件同套概率）。
- `BIAS_CHANCE = 80`（战后候选偏向主套概率）。
- 起始/招募发件数 `INITIAL_GRANT = 3`；战后候选数 `BATTLE_ROLL = 8`、可选上限 `BATTLE_PICK = 2`。
- 套装阈值 `SET_TIERS = [3,6,9]`。
- 每套件数 `SET_SIZE = 9`、套装数 `SET_COUNT = 8`。
- 稀有度配色表、每套 9 件的稀有度/数值梯度表（生成器输入）。

## 8. 验收标准（Acceptance Criteria）

- **AC-1（数据）**：装备池含 8 套 × 9 槽 = 72 件，每件 `set_id` 合法、`slot` 唯一覆盖该套 9 槽、`id` 全局唯一；`EquipmentDataManager` 扫描零校验错。
- **AC-2（直发）**：`start_run` 后每名起始船员、`confirm_recruit` 后新兵各持 **3 件不同槽**；固定 seed 下 80% 分支可复现（断言具体结果或边界）。
- **AC-3（战后滚选）**：非末岛胜利进入 `RUN_EQUIPPING`，`_pending_battle_equip` 含全体出战存活者各 8 件；`equip_piece` 正确处理空槽装/替换/cap9；选满或跳过后转 `RUN_RECRUITING`。
- **AC-4（偏向）**：固定 seed 下，有主套船员的 8 件候选中主套件占比符合 80% 倾向（断言计数/具体序列）；无主套者全随机。
- **AC-5（计数器）**：`get_set_counts`/`get_dominant_set`（含并列字典序）/`get_active_set_tier`（0/3/6/9 边界）正确。
- **AC-6（存档）**：`_pending_battle_equip` + `RUN_EQUIPPING` 阶段往返一致；旧档（无新键）优雅加载；`roster_equipment` 兼容 ②a。
- **AC-7（UI，ADVISORY）**：F5 下战后补装屏渲染 8 候选 + 人形纸娃娃（9 槽名字/稀有度色/套装标签/激活档位），选 2/替换确认可用。
- **AC-8（无副作用）**：本期不引入任何战斗加成；`BattleResolution`/`UnitInstance.get_*` 行为不变；全量测试零错误/零孤儿。

## 9. 非目标（本期不做，留 ②b-2/3）

- 任何套装**战斗加成效果**（减伤/附加伤害/自愈/吸血/反伤/斩杀/控制）。
- 套装效果应用框架、per-round 钩子、execute_attack 钩子、寒霜 debuff。
- 装备 UI 美术资源（本期纯白盒）、职业限装、跨 run 装备持久（本期仍随 run 存在/阵亡消失）。
- 选航战利品经济（②c）。
